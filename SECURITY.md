# Security Policy

## Reporting

Please do not open public GitHub issues for suspected security vulnerabilities.

Report them privately through the AItoAir contact form below and include
`FramePin security report` in the message:

https://aitoair.com/en/contact-us-en/

## Scope reminders

Security reports are especially helpful when they involve:

- authentication and session handling
- authorization and team isolation
- file upload and export handling
- deployment defaults
- accidental exposure of private assets, checkpoints, or datasets

## Disclosure expectations

- Give the maintainers time to investigate and patch the issue.
- Avoid publishing proof-of-concept exploit details before a coordinated fix is
  available.
